# MiraKuru Human Model

## 1. 目的

MiraKuruは、若者の発話を単純な感情記録や強み診断に落とし込まない。

扱いたいのは、次の流れである。

```text
日々の体験
-> その日の状態
-> 心が動いた反応
-> 何から守られていたか
-> 根底の欲求
-> 外に出る表れ方
-> 小さな一歩
```

このモデルは、心理学、行動科学、脳科学の知見を、MiraKuru向けに簡略化した内部設計である。

## 2. 学問から借りる軸

### 自己決定理論

使う要素:

- 自律性
- 有能感
- 関係性

MiraKuruでの言い換え:

- 自分で選びたい
- できるようになりたい
- つながっていたい

役割:

- 根底の欲求を読む土台にする
- 「この人が何を求めて反応したか」を整理する

### ナラティブ・アイデンティティ

使う要素:

- 体験を意味づけし、自分の物語にまとめる

MiraKuruでの言い換え:

- 点が線になる
- 前にも出てきた反応が見える

役割:

- Graphの考え方の核にする
- 1回ごとの会話を、蓄積のある自己理解に変える

### 感情の粒度

使う要素:

- 感情を細かく言い分けられるほど、扱いやすくなる

MiraKuruでの言い換え:

- 心が動いたところ
- 何が少し重かったか

役割:

- 「つらい」「不安」だけで終わらせず、もう少し細かく見る
- 返答を押しつけずに深める

### 体験回避と心理的柔軟性

使う要素:

- 行動には、何かから自分を守る役割があることがある

MiraKuruでの言い換え:

- その時間が何から守ってくれていたか

役割:

- スマホ、ゲーム、ぼーっとした時間を、単純な悪習慣として扱わない
- 寄り添いの土台にする

### 実行意図

使う要素:

- 抽象的な目標より、具体的な小さい行動の方が動きやすい

MiraKuruでの言い換え:

- 小さな一歩
- 10分版
- 1つだけやる

役割:

- 返答を「良い話」で終わらせず、軽い行動に変える

### 報酬予測誤差

使う要素:

- 予想より少し良かった体験が、学習と継続につながる

MiraKuruでの言い換え:

- 少し見えた
- 少し進めた
- ここまで残せた

役割:

- 毎回大きな変化を求めない
- 続けたくなる小さな達成感を作る

## 3. MiraKuruの内部モデル

MiraKuruは、ユーザーを「診断」するのではなく、以下の項目を毎回仮置きする。

### 3.1 保存する最小項目

- 今日の点
- 素材
- 状態
- 心が動いたところ
- 守っていたもの
- 根底の欲求
- 表れ方
- 小さな一歩

### 3.2 各項目の意味

#### 今日の点

ユーザーがその日に置いた短文。

例:

- ゲーム実況を見ていた
- スマホを見すぎて罪悪感があった
- 将来が怖い

#### 素材

その話が何についてのものか。

例:

- ゲーム実況
- 裁縫
- 料理動画
- 進路不安
- 親との会話

趣味や体験は、才能の原石の一部として扱う。

#### 状態

その日にどれくらい余力があるか。

例:

- 休息優先
- 受け止め優先
- 少し見られる
- 少し動ける

状態が悪い日は、強み化や制作提案を急がない。

#### 心が動いたところ

何に反応したか。

例:

- 安心した
- 面白かった
- 見続けてしまった
- わかりやすかった
- しんどかった

#### 守っていたもの

その行動や時間が、何から守ってくれていたか。

例:

- 退屈
- 不安
- 孤独感
- 考えすぎ
- 責められる感覚

これは、寄り添いのための重要項目であり、無理に毎回断定しない。

#### 根底の欲求

その人が何を求めていたか。

初期ラベル:

- 自分で選びたい
- できるようになりたい
- つながっていたい
- 落ち着きたい
- 気になってしまう
- 役に立ちたい

#### 表れ方

その欲求が、どのように外に出ているか。

初期ラベル:

- 気づく
- 深める
- つなぐ
- 整える
- 作る
- 届ける

#### 小さな一歩

今の状態でできる最小単位の行動。

初期ラベル:

- 観察する
- 言葉にする
- 小さく作る
- 1人に試す

## 4. 才能の原形の定義

MiraKuruでは、才能を固定ラベルとして断定しない。

扱うのは、次の式である。

```text
才能の原形 = 素材 × 根底の欲求 × 表れ方
```

例:

```text
素材: ゲーム実況
根底: つながっていたい
表れ方: 気づく

表示文:
人がひとりじゃない感じを受け取れる見せ方に、
反応しやすいかもしれない。
```

```text
素材: 裁縫
根底: 自分で選びたい
表れ方: 作る

表示文:
自分のイメージを、小さく形にして確かめる方向で
力が出やすいかもしれない。
```

## 5. ユーザーに見せるもの

内部で全項目を持っていても、毎回全部見せない。

MVPで見せるのは次の4つに絞る。

- 今日のこと
- 心が動いたところ
- 君の中にありそうな原形
- 小さな一歩

以下は内部中心で扱う。

- 状態
- 守っていたもの
- 根底の欲求
- 表れ方

これらは自然文に変換して、必要な時だけ表に出す。

## 6. Graphの役割

MiraKuru Graphは、スキル診断ではなく、点と線の地図である。

MVPでは、Graphに次のような線を作る。

- 同じ素材が何度も出る
- 同じ根底の欲求が別の素材でも出る
- 同じ表れ方が別の場面でも出る
- 同じ「守っていたもの」が繰り返し出る

例:

```text
ゲーム実況
学校の説明
料理動画
```

この3つで「わかりやすさ」への反応が重なれば、
説明の伝わりやすさに感度がある、という線の候補になる。

## 7. `character` フォルダの再利用方針

対象フォルダ:

- `/Users/kusakariakiraakira/Desktop/character`

### 再利用できるもの

- 質問から内部ラベルへ変換する構造
- 結果を自然文にする構造
- 途中保存と復元
- 共有用の出力導線

具体的には次のファイルが使える。

- `compute.js`
- `render.js`
- `quiz.js`
- `main.js`
- `share.js`

### 再利用しないもの

- Big Five / VIA のそのままの診断軸
- 40問の長尺診断
- 科学的診断アプリとしての見せ方

### MiraKuruへの転用イメージ

```text
Big Five / VIA
-> MiraKuruの内部ラベルへ置換

質問回答
-> 今日の点 + 反応 + 守っていたもの

スコア計算
-> 仮置きの原形生成

診断結果画面
-> 今日のまとめ + Graph予告
```

## 9. 現時点の方針

MiraKuruは、若者の言葉をすぐに強みへ変換するアプリではない。

まず、

- 今日はどんな状態だったか
- 何に反応したか
- 何から守られていたか

を見たうえで、

- その中にどんな原形がありそうか
- 今の状態で何なら動けるか

を仮置きするアプリである。
