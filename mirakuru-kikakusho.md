# MiraKuru 企画書

## 1. コンセプト

**MiraKuru は、スクリーンの中で過ごした時間を、未来につながる小さな一歩に変えるAIアプリです。**

YouTube、ゲーム、SNS、日々の出来事。外から見ると「ただ見ていた」「ただ遊んでいた」「何もしていない」に見える時間の中にも、本人だけの興味、違和感、憧れ、不満、観察、学びの種がある。

MiraKuru は、その一次体験をAIとの対話で言語化し、スキル、課題、制作案、未来の選択肢へ変えていく。

**タグライン案**

- 今日の好きから、未来がくる。
- 減らすんじゃない。つなげる。
- スクリーンの時間を、未来の種に。
- 止まって見える日にも、未来は来る。

## 2. 背景

AI時代には、知識そのものの価値は相対的に下がっていく。AIに聞けば多くの答えは得られる。

一方で、人間の価値として残りやすいのは、次の3つである。

- **一次体験**: 自分が実際に見た、感じた、困った、試したこと
- **スキル**: 体験を通じて育った観察力、言語化力、制作力、対人理解など
- **文脈**: なぜそれに惹かれたのか、誰のどんな課題につながるのか

しかし多くの若者は、自分の体験を「価値」として扱う方法を知らない。

特に、不登校、進路不安、将来不安、自己肯定感の低下がある人にとって、YouTubeやゲームやSNSに時間を使うことは、外からは「逃げ」や「浪費」に見えやすい。

でも本人の中では、未来への不安を抱えながら、何かに惹かれ、何かを探し、何かに安心し、何かに違和感を持っている。

MiraKuru は、その時間を否定しない。

**今いる場所から、未来へ一本線を引く。**

## 3. 解決したい課題

### ユーザーの課題

- 自分の好きなことが将来にどうつながるかわからない
- YouTubeやゲームばかりしている自分に罪悪感がある
- 将来が不安だが、何から始めればいいかわからない
- 自分にはスキルや実績がないと思っている
- 起業や制作に興味はあるが、アイデアの出し方がわからない
- ChatGPTを使えばいいと言われても、何を聞けばいいかわからない
- 日記や習慣アプリは続かない

### 既存サービスの課題

- 日記アプリは記録で終わりやすい
- 習慣アプリは継続できない人を責める構造になりやすい
- スクリーンタイム系アプリは「制限」「減らす」に寄りやすい
- キャリアアプリは不安の強い若者には眩しすぎる
- 汎用AIは単発回答は強いが、体験の蓄積、可視化、再開導線が弱い

## 4. ターゲット

### 初期ターゲット

**不登校・進路不安・将来不安があり、YouTube、ゲーム、SNSに時間を使っているが、本当はこのままでいいのか不安な若者。**

### 拡張ターゲット

- 起業したい学生
- 自分の強みを見つけたい若手社会人
- 子どものスマホ時間を責めずに未来へつなげたい親
- 探究学習やキャリア教育に関わる学校、塾、支援施設
- AIを使って何か作りたいが、テーマが見つからない人

## 5. プロダクトの中核体験

MiraKuru の基本フローは、次の5ステップ。

1. **記録する**
   - 今日見たもの、遊んだもの、気になったこと、困ったことを1つ書く
   - 例: 「ゲーム実況を3時間見た」「TikTokで料理動画をずっと見た」

2. **好き・違和感を掘る**
   - AIが責めずに聞く
   - 「どこが面白かった？」「不満はある？」「自分ならどう変えたい？」

3. **構造化する**
   - AIが一次体験を分解する
   - 好きな要素、隠れた欲求、不満、関連スキル、未来候補を抽出

4. **小さく作る**
   - 10分、30分、週末でできる制作案を出す
   - 例: 「好きな実況者の見やすい理由を3つ書く」「30秒台本を作る」

5. **未来の地図にする**
   - 記録がスキルグラフ、興味マップ、週次レポートに蓄積される
   - 「あなたは説明、編集、ゲーム理解に反応している」などを返す

## 6. 代表的な利用シーン

### YouTubeばかり見ている若者

入力:

> ゲーム実況をずっと見ていた。

AI:

> それはただの消費に見えるかもしれないけど、心が動いた理由があるはず。  
> どこが一番よかった？ 話し方、編集、攻略、友達感、リアクション、コメント欄、どれが近い？

深掘り後の出力:

- 好きな要素: テンポの良い説明、失敗しても笑える空気、攻略のわかりやすさ
- 隠れたスキル: 観察力、ゲーム理解、説明への感度、編集感覚
- 小さな制作案: 好きな実況者の「見やすい理由」を3つ書く
- 未来候補: 動画編集、ゲーム企画、配信サポート、コミュニティ運営

### 起業したい学生

入力:

> 友達が課題の締切をよく忘れている。

AI出力:

- 課題: 学校、バイト、サークルで予定が散らばる
- 顧客候補: 大学生、専門学生、高校生
- 小さな解決案: LINEで課題を共有できるミニBot
- 検証: 友達3人に1週間使ってもらう
- 必要スキル: 通知設計、ヒアリング、簡単な自動化、UX設計

### 親子利用

入力:

> 子どもがずっとMinecraftをしていた。

AI出力:

- 好きな要素: 建築、探索、収集、協力
- 学びの種: 空間認識、計画、試行錯誤、資源管理
- 親子の会話案: 「今日作ったもののこだわりを1つ教えて」
- 小さな制作案: 作った建物の設計図を紙に描く

## 7. MVP

**機能**

- 1日1つの体験入力
- AIによる深掘り質問
- AIによる構造化

  - 不満
  - 隠れた興味
  - 関連スキル
  - 小さな制作案
  - 未来候補

### MVPで作らないもの

- 複雑なSNS機能

## 8. ガチモード

### Screen Insight

ユーザーが許可したスクリーンタイム情報、または手入力の利用履歴をもとに、アプリごとの傾向を深掘りする。

目的は制限ではなく、**好きの構造化**。

例:

- YouTubeをよく見る
  - 何の動画を見る？
  - どの瞬間に手が止まる？
  - どんな人の話し方が好き？
  - 不満はある？
  - 自分なら何を作りたい？

- TikTokをよく見る
  - どんな投稿を保存する？
  - 何に憧れる？
  - 何が見つけづらい？
  - その不満を解決する小さなツールは作れる？

- ゲームをよくする
  - 何が気持ちいい？
  - 成長、収集、対戦、協力、世界観、操作感のどれ？
  - 面倒なところは？
  - その不満はゲーム企画やUI改善につながる？

### ガチモードの出力

- アプリ別の好きな要素
- アプリ別の不満
- 隠れた欲求
- 関連する職業
- 作れるミニプロジェクト
- 検証方法
- 必要な学習テーマ

## 9. 人間メンター構想

最初から実装しないが、将来的に強い機能。

### Phase 1: AI職業インタビュー

AIが、動画編集者、ゲームプランナー、起業家、デザイナー、エンジニアなどの視点で質問に答える。

### Phase 2: 実在インタビュー推薦

ユーザーの興味に合う職業インタビュー、会社ブログ、YouTube、note記事を推薦する。

### Phase 3: 非同期Q&A

現役の人に質問を送れる。

- 1質問単位
- 事前にAIが質問を整理
- 回答後にAIが学びと次の一歩を要約

### Phase 4: 15分メンターチャット

現役の人と短時間で話す。

- AIが事前質問を整理
- 終了後に学び、関連スキル、次の制作案を生成
- 未成年対応、安全性、審査、録音許可などを慎重に設計する

## 10. 既存AIとの違い

ChatGPTでも単発では似たことができる。

しかし MiraKuru の価値は、AIの回答そのものではなく、体験設計と蓄積にある。

| 既存AI | MiraKuru |
| --- | --- |
| ユーザーが毎回質問を考える | アプリが問いを届ける |
| 単発の回答が中心 | 体験が蓄積される |
| 会話履歴に埋もれる | スキル、課題、制作案として構造化される |
| 何でもできるが続けにくい | 10秒入力、再開しやすい設計 |
| 答えを出す | 一次体験から未来の種を見つける |
| 汎用チャット | 若者の未来接続に特化した体験 |

一言で言うと、

**ChatGPTは道具。MiraKuruは、一次体験を未来に変える習慣と資産化の仕組み。**

## 12. 安全性と倫理

MiraKuru は、若者の不安や孤独に触れる可能性があるため、安全設計を最初から重視する。

### 原則

- ユーザーを責めない
- スマホ時間を一方的に悪としない
- 「頑張れ」「大丈夫」と簡単に言わない
- 他者の権利を侵害する制作や販売へ誘導しない
- 医療・心理支援の代替を名乗らない
- 深刻な危機には専門窓口を案内する
- 未成年のデータ保護を最優先する
- 保護者や学校向け機能では本人の尊厳と同意を重視する

### トーン

MiraKuru は怒らない。焦らせない。決めつけない。  
でも、未来をあきらめない。

### 寄り添いの思想

不登校や強い不安の中にいる人にとって、「頑張れ」「大丈夫」「きっとできる」は、ときに重すぎる言葉になる。

本人はすでに頑張っているかもしれない。大丈夫ではないから苦しいのかもしれない。未来を考えたいのに、考えるほど怖くなる状態かもしれない。

だから MiraKuru は、安易な励ましではなく、まず今の状態をそのまま受け止める。

使う言葉:

- 今日はここに書けただけで十分
- まだ答えを出さなくていい
- 今すぐ変わらなくていい
- その時間にも、何か心が動いたものがあったかもしれない
- 一緒に、ほんの少しだけ見てみよう

避ける言葉:

- 頑張ろう
- 大丈夫
- もっと努力しよう
- 早く行動しよう
- みんなできている
- 将来のために今やらなきゃ

MiraKuru の寄り添いは、前向きにさせることではなく、**前を向けない日にも隣にいること**から始まる。

### 制作・販売の安全ルール

MiraKuru は、ユーザーが好きな作品、ゲーム、動画、キャラクターから影響を受けて制作することを肯定する。

ただし、既存作品のキャラクター、ロゴ、アイテム、イラスト、名称などをそのまま使って販売する提案はしない。特にメルカリ、BOOTH、minne、イベント販売などに出す場合は、権利侵害につながる可能性がある。

代わりに、次のような安全な方向へ案内する。

- 好きな作品から受けた感情や雰囲気を、自分のオリジナルに変える
- キャラクターそのものではなく、「冒険っぽい」「回復アイテムっぽい」「森の探索っぽい」など抽象化する
- まずは販売ではなく、自分用、写真記録、家族や友人に見せるところから始める
- 販売する場合は、オリジナル作品として出せるかを確認する

MiraKuru は、創作意欲を止めない。  
でも、知らないうちに危ない橋を渡らないように、やわらかく安全な作り方へ導く。

## 13. 初期検証

### 検証したい仮説

1. YouTubeやゲーム時間を責めずに扱うと、若者は本音を入力しやすい
2. 「好き」「不満」「作れるもの」への分解は、未来不安の軽減につながる
3. 10分制作案は、行動のハードルを下げる
4. 週次レポートは、自己理解と継続に効く
5. 起業志望者は、日常の違和感から事業アイデアを作る用途に価値を感じる

### 最初に聞く質問

- このアプリは説教くさく感じるか？
- 入力した後、少し未来につながった感覚があるか？
- AIの返答は自分を見てくれている感じがあるか？
- 10分制作案は実際にやれそうか？
- 週次レポートを見たいと思うか？
- お金を払うなら、どの機能か？

## 14. 最初の実装案

### 技術構成

- Webアプリから開始
- 認証: メールまたは匿名ログイン
- DB: Supabase など
- AI: OpenAI API
- フロント: Next.js
- 通知: 最初はメールまたはLINE Bot風、その後PWA通知

### 主要データ

```json
{
  "entry": "ゲーム実況を3時間見た",
  "source_type": "youtube",
  "emotion": ["安心", "憧れ", "退屈しのぎ"],
  "liked_elements": ["話し方", "編集", "攻略"],
  "frictions": ["自分では作れない気がする"],
  "hidden_interests": ["動画編集", "ゲーム理解", "説明"],
  "skills": ["観察力", "言語化", "構成理解"],
  "micro_projects": [
    "好きな実況者の見やすい理由を3つ書く",
    "30秒の実況台本を作る",
    "自分なら変えたいゲームUIを1つ描く"
  ],
  "future_paths": ["動画編集", "ゲーム企画", "配信サポート"]
}
```

## 16. まとめ

MiraKuru は、若者に「ちゃんとしろ」と言うアプリではない。

YouTubeを見ていた日も、ゲームをしていた日も、SNSを眺めていた日も、その中にあった小さな反応を拾い上げる。

そして、それを好き、不満、問い、スキル、制作案、未来候補へ変えていく。

**止まって見える時間にも、未来への入口がある。**

MiraKuru は、その入口を一緒に見つけるアプリである。
